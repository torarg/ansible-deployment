CLI Reference

ansible-deployment

All available commands are listed below.

Each command has it’s own help that can be shown by passing ‘–help’:

$ ansible-deployment COMMAND –help

ansible-deployment [OPTIONS] COMMAND [ARGS]...

Options

--version

Show the version and exit.

--debug

Enable debug output.

commit

Commit all changes.

ansible-deployment commit [OPTIONS]

delete

Delete deployment.

Deletes all created files and directories in deployment directory and also purges deployment data from configured inventory writers.

If –from-writer is specified the deployment will only be deleted from the specified inventory writers. The deployment directory will NOT be deleted if this option is used.

ansible-deployment delete [OPTIONS]

Options

-w, --from-writer <from_writer>

Only delete deployment from specified inventory writer. May be specified multiple times.

--non-interactive

Don’t ask before deleting deployment.

diff

Show deployment diff.

ansible-deployment diff [OPTIONS]

edit

Edit deployment with vim.

ansible-deployment edit [OPTIONS]

fetch-key

Fetch deployment key from given inventory source.

ansible-deployment fetch-key [OPTIONS] INVENTORY_SOURCE

Arguments

INVENTORY_SOURCE

Required argument

init

Initialize deployment directory.

Initialization requires a ‘deployment.json’ file in the current working directory.

ansible-deployment init [OPTIONS]

Options

--non-interactive

Don’t ask before initializing deployment.

lock

Encrypt all deployment files except the roles directory.

ansible-deployment lock [OPTIONS]

log

Show deployment log.

ansible-deployment log [OPTIONS]

pull

Pull encrypted deployment repo.

ansible-deployment pull [OPTIONS]

push

Run configured ìnventory_writers and push encrypted git repo.

ansible-deployment push [OPTIONS]

Options

--template-mode

Run inventory writers without ssh and deployment keys.

--force

Force push.

reset

Hard reset deployment to last commit.

ansible-deployment reset [OPTIONS]

run

Run deployment with ansible-playbook.

ansible-deployment run [OPTIONS] [ROLE]...

Options

-l, --limit <limit>

Limit playbook execution.

-e, --extra-var <extra_var>

Set extra var for playbook execution.

-d, --disable-host-key-checking

Disable ssh host key checking.

Arguments

ROLE

Optional argument(s)

show

Show deployment information.

Deployment information may be filtered by specifying attribute(s).

ansible-deployment show [OPTIONS] [ATTRIBUTE]...

Arguments

ATTRIBUTE

Optional argument(s)

ssh

Run ‘ssh’ command to connect to a inventory host.

ansible-deployment ssh [OPTIONS] HOST

Arguments

HOST

Required argument

unlock

Decrypt all deployment files except the roles directory.

ansible-deployment unlock [OPTIONS]

Options

--force

Unlock even if deployment verification failed.

update

Update deployment.

Updates deployment inventory from inventory_sources and pulls changes from roles repository.

If –from-source is specified the roles repository will still be updated but only the given inventory sources will be queried for updates.

ansible-deployment update [OPTIONS]

Options

--non-interactive

Apply all updates without asking.

-s, --from-source <from_source>

Only query given inventory sources. May be specified multiple times.

update-known-hosts

Force update of known_hosts file.

ansible-deployment update-known-hosts [OPTIONS]
